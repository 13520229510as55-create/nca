## ⚠️ 重要：GCP Service Account 凭证

NCA Toolkit 需要 GCP Service Account 凭证来访问 Google Cloud Storage。

### 选项 1：从现有服务获取（推荐）

如果您已经有运行中的 NCA Toolkit 服务，可以获取现有凭证：

```bash
# 获取现有服务的环境变量
gcloud run services describe <existing-service-name> \
  --region=us-central1 \
  --format="value(spec.template.spec.containers[0].env)" | grep GCP_SA_CREDENTIALS
```

### 选项 2：创建新的 Service Account

如果还没有，需要创建：

```bash
# 1. 创建 Service Account
gcloud iam service-accounts create nca-toolkit-sa \
  --display-name="NCA Toolkit Service Account"

# 2. 授予 Storage 权限
gcloud projects add-iam-policy-binding gen-lang-client-0960629066 \
  --member="serviceAccount:nca-toolkit-sa@gen-lang-client-0960629066.iam.gserviceaccount.com" \
  --role="roles/storage.admin"

# 3. 创建密钥
gcloud iam service-accounts keys create key.json \
  --iam-account=nca-toolkit-sa@gen-lang-client-0960629066.iam.gserviceaccount.com

# 4. 获取 JSON 内容（需要添加到部署脚本）
cat key.json
```

### 选项 3：使用 Cloud Run 默认服务账户

如果您的 Cloud Run 服务账户已经有 GCS 权限，可能不需要单独设置。
