## ⚠️ 关于 GCP Service Account 凭证

如果部署过程中提示需要 `GCP_SA_CREDENTIALS`，您需要：

### 选项 A：从现有服务获取（如果您已有 NCA 服务）

```bash
# 列出现有的 Cloud Run 服务
gcloud run services list --region=us-central1

# 获取现有服务的 GCP_SA_CREDENTIALS
gcloud run services describe <existing-service-name> \
  --region=us-central1 \
  --format="get(spec.template.spec.containers[0].env)" | grep GCP_SA_CREDENTIALS
```

### 选项 B：使用 Cloud Run 默认服务账户

如果您的 Cloud Run 默认服务账户已经有 GCS 权限，可能不需要单独设置。
