```bash

REGION="asia-east1"                      # 改为您的 Cloud Run 区域
SERVICE_NAME="nca-toolkit"               # Cloud Run 服务名称（可保持默认）
API_KEY="your-api-key"                   # 改为您的 NCA API Key
GCP_BUCKET_NAME="your-bucket-name"       # 改为您的 GCS Bucket 名称
```

## 🎨 使用中文字体

### 在 n8n 工作流中配置

```json
{
  "video_url": "视频URL",
  "settings": {
    "font_family": "Noto Sans SC",  // 简体中文（推荐）
    "word_color": "#FFFFFF",
    "font_size": 40,
    "bold": true,
    "position": "bottom_center",
    "style": "highlight"
  }
}
```

### 可用字体名称

- `Noto Sans SC` - 简体中文（推荐）⭐
- `Noto Sans TC` - 繁体中文
- `WenQuanYi Zen Hei` - 文泉驿正黑
- `WenQuanYi Micro Hei` - 文泉驿微米黑

## ⚠️ 注意事项

1. **构建时间**：首次构建需要 30-40 分钟
2. **镜像大小**：添加字体会增加约 200-300MB
3. **GCP 凭证**：确保已配置 GCP Service Account 凭证
4. **API Key**：使用强密码作为 API Key

## 🔧 故障排查

### 问题 1：构建失败

**检查**：
- Dockerfile 语法是否正确
- 网络连接是否正常
- GCP 权限是否足够

### 问题 2：部署失败

**检查**：
- GCP 项目 ID 是否正确
- API Key 是否有效
- GCS Bucket 是否存在

### 问题 3：中文仍显示方块

**检查**：
- 字体名称是否正确（区分大小写）
- 是否使用了 `Noto Sans SC` 而不是 `noto`
- 查看 Cloud Run 日志确认字体是否安装
